Obsidian + Hexo 完美维护个人博客

本文最后更新于 2025-12-06 19:00:19

我从 22 年 7 月底开始接触 Obsidian，用它维护我的笔记本有 3 年多的时间了。一直构想搭建一个博客网站，在笔记本里写点东西发到网上。

我的需求是：本地继续使用 Obsidian 维护笔记仓库，写完笔记以后能够直接通过几行命令发布博客，不要配置图床。

最近摸索出了一套对我来说几乎完美的零成本解决方案：使用 Obsidian 撰写维护本地笔记，使用 Hexo 构建网页，上传到 Github 发布。

  1. Obsidian 本地撰写笔记
  2. Hexo 将笔记转换成网页文件，并上传到 Github
  3. Github 将网页文件渲染成网页

Obsidian 配置

Hexo 的博客文件夹结构如下图所示
所有博客笔记都以 .md 的格式存放在 /source/_posts 文件夹下
这里设置 source 文件夹作为 Obsidian 的工作文件夹打开

Obsidian 按下图进行设置

  1. 新笔记文件总是创建在 _posts 文件夹中
  2. 建立一个文件夹，比如 attachments 用来存放笔记中的图片
  3. 将笔记中的图片链接形式修改成相对路径
  4. 取消 Obsidian 图片的特殊链接格式，改成用标准的 Markdown 格式链接

经过上述设置后，在 Obsidian 中插入的图片会自动进入 attachments 文件夹，并在笔记中以标准 Markdown 格式插入，如 ![](../attachments/xxx.png) 这样上传网页的时候不需要做任何的修改，就可以正确显示图片。

上述全部设置完成以后，更新博客的工作流程就相当简单了

  1. 使用 Obsidian 打开 source 库，写博客
  2. 打开 cmd 到博客存放路径，输入几行命令
    1. hexo clean (清理缓存)
    2. hexo g (生成网页文件)
    3. hexo s (本地部署，检查有无问题)
    4. hexo d (上传发布网页)
  3. 等待几分钟，博客网站就更新好了
